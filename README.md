## 1. 项目简介

这是一个从零实现的 **LLaMA 风格 Decoder-only Transformer** 模型，包含了现代大语言模型（LLM）的几大核心技术：

| 技术 | 说明 |
| ------ | ------ |
| **RMSNorm** | LLaMA 采用的归一化方法，比 LayerNorm 更简单高效 |
| **RoPE**（旋转位置编码） | 给每个词打上"位置标签"，让模型理解语序 |
| **GQA**（分组查询注意力） | 省显存的注意力机制，推理时 KV 缓存减少数倍 |
| **SwiGLU** | 带门控的前馈网络，比传统 ReLU 效果更好 |
| **残差连接** | 让深层网络训练更稳定的关键技巧 |
| **BPE 分词器** | 自动学习如何把文字切成模型能理解的数字 |

## 3. 为什么要这样设计？——核心概念通俗版

在深入公式之前，先用一个简单的比喻理解整个模型：

> **把模型想象成一个人阅读文章的过程：**
>
> 1. **分词器** = 识字卡片。把 "今天天气真好" 切成一个个小块，每块给个编号。
> 2. **Embedding** = 查字典。每个编号对应一组 768 个小数的"含义向量"。
> 3. **Attention（注意力）** = 开会讨论。读到 "吃了" 时，模型会回头看 "谁吃了？"，自动关注相关的词。
> 4. **MLP（前馈网络）** = 独自消化。交换完信息后，每个人回工位思考整理。
> 5. **堆叠 N 层** = 反复揣摩。一层不够，再来一层，12 层之后理解就很深了。
> 6. **输出层** = 预测下一个字。读完 "今天天气真" 后，猜下一个字，得分最高的就是 "好"。

## 4. 模型架构详解

### 4.1 整体数据流

```shell
输入文本： "今天天气"
    │
    ▼
┌─────────────┐
│  Tokenizer  │  把文字切成编号： [234, 567, 890]
└──────┬──────┘
       ▼
┌─────────────┐
│  Embedding  │  查表，每个编号变成 dim=768 维的向量
└──────┬──────┘
       ▼
┌─────────────┐
│  Dropout    │  训练时随机丢弃一些信息，防止死记硬背
└──────┬──────┘
       ▼
╔═══════════════════════════════════════════╗
║         DecoderLayer × 12 层              ║
║                                           ║
║   每层内部：                               ║
║     x ─→ [RMSNorm] ─→ [Attention] ─→ +x  ║
║       ─→ [RMSNorm] ─→ [MLP]      ─→ +x  ║
║                                           ║
╚══════════════╤════════════════════════════╝
               ▼
       ┌──────────────┐
       │   RMSNorm    │  最终归一化
       └──────┬───────┘
              ▼
       ┌──────────────┐
       │   Linear     │  输出每个字的得分 (vocab_size=6144 个分数)
       └──────┬───────┘
              ▼
       logits: [batch, seq_len, 6144]
```

### 4.2 RMSNorm — 数值稳定器

**作用**：把数据压在一个合理的范围内，防止经过多层计算后数值爆炸或消失。

**公式**：

```shell
RMSNorm(x) = x / sqrt(mean(x²) + ε) × γ
```

**通俗解释**：

- 先算每个向量所有元素的"平均能量"（均方根 RMS）
- 用原始值除以这个能量 → 把数值范围压到 1 左右
- 再乘以一个可学习的缩放因子 γ，给模型留调整空间
- ε = 1e-5 是防止除以 0 的小常数

**和 LayerNorm 的区别**：

| | LayerNorm | RMSNorm |
| --- | --- | --- |
| 要做的事 | 先减均值，再除标准差 | 只除 RMS，不减均值 |
| 计算量 | 大 | 约省 30% |
| LLaMA 在用？ | ❌ | ✅ |

### 4.3 注意力机制 — 模型的"眼睛"

注意力机制让每个词能够"看到"句子中的其他词，并根据相关性自动决定关注谁。

#### 核心公式

```shell
Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) × V
```

**一步一步拆解**：

1. **生成 Q, K, V**：每个词通过三个不同的矩阵变换，产生三个向量
   - **Q（Query，查询）**："我是 '苹果' 这个词，我想知道前面谁跟我有关"
   - **K（Key，标签）**："我是 '吃了' 这个词，我身上贴着 '动作' 的标签"
   - **V（Value，内容）**："我是 '吃了' 这个词，我的实际含义是 xxx"

   ```shell
   Q = x · Wq,    K = x · Wk,    V = x · Wv
   ```

2. **计算相关性分数**：Q × K^T，看"查询"和"标签"的匹配程度

   ```shell
   scores = Q·Kᵀ / √dₖ
   ```

   除以 √d_k 的原因：向量维度越大，点积结果越大，会导致 softmax 后梯度消失，除以 √d_k 可以缓解。

3. **softmax 归一化**：把分数变成概率（总和为 1）

   ```shell
   αᵢⱼ = exp(scoreᵢⱼ) / Σₖ exp(scoreᵢₖ)
   ```shell
   αᵢⱼ 表示 "第 i 个词对第 j 个词的关注程度"。

4. **加权求和**：用注意力权重去提取 V 的信息

   ```shell
   output = α · V
   ```

5. **因果遮罩（Causal Mask）**：生成式模型的特殊规则——每个词只能看它自己和前面的词，不能"偷看"未来。（上三角矩阵设为 -∞，softmax 后变成 0）

#### 技术一：GQA（分组查询注意力）— 省显存

标准的多头注意力（MHA）：Q 有几个头，K/V 就有几个头，一对一套。

GQA 的做法：**让 K/V 的头数少于 Q 的头数，多个 Q 头共享一组 K/V 头**。

```shell
MHA:   Q [头0][头1][头2][头3]    K [头0][头1][头2][头3]    V [头0][头1][头2][头3]
GQA:   Q [头0][头1][头2][头3]    K [头A      ][头B      ]    V [头A      ][头B      ]
              ↑────┬────↑              ↑              ↑
              头0~1 共享 K/V-A         头2~3 共享 K/V-B
```

- n_kv_heads = n_heads  → 退化为 MHA（效果最好，显存最大）
- n_kv_heads = 2~8      → GQA（效果接近 MHA，显存省数倍）
- n_kv_heads = 1        → 退化为 MQA（最省显存，效果略差）

KV 缓存大小公式：

```shell
KV Cache Size = 2 × n_layers × n_kv_heads × d_head × seq_len
```

---

#### 技术二：RoPE（旋转位置编码）— 给词打"位置标签"

**为什么需要位置编码？**
"饭吃我" 和 "我吃饭" 字完全一样，但不加位置信息，模型眼里它们一模一样。

**RoPE 的思路**：

- 把每个词的向量按相邻两个维度配对，看成 2D 向量
- 每个位置 pos 对应一个旋转角度 θ = f(pos)，把向量旋转这个角度
- 两个词做点积时，天然包含它们的相对位置差

**频率公式**（决定每个维度转多快）：

```shell
θᵢ = 1 / 10000^(2i/d)
```

其中 i 是维度对的索引，d 是头维度。

- 高频维度（i 小）→ 转得快 → 捕捉精细的局部关系
- 低频维度（i 大）→ 转得慢 → 捕捉大范围的长距离关系

**角度矩阵**：

```shell
Θ_pos = pos × [θ₀, θ₁, ..., θ_{d/2-1}]    （一行向量，长度 = d/2）
```

**旋转公式**（2D 旋转矩阵）：

```shell
[ x_new_r ]   =   [ cosθ  -sinθ ]   [ x_r ]
[ x_new_i ]       [ sinθ   cosθ ]   [ x_i ]
```

即：

```shell
x_new_r = x_r·cosθ - x_i·sinθ
x_new_i = x_r·sinθ + x_i·cosθ
```

**为什么只转 Q 和 K，不转 V？**

- 位置信息通过 Q × K 点积引入：Q 转了角度 pos_m，K 转了角度 pos_n
- 点积结果 ≈ cos(pos_m - pos_n) × 内容相似度 → 自动包含相对位置
- V 存的是"实际内容"，不需要再带位置信息

#### Flash Attention — 高速版本

项目默认启用 PyTorch 内置的 `scaled_dot_product_attention`，它是 Flash Attention 的高效实现：

- **原理**：利用 GPU 的 SRAM（快但小）做分块计算，减少 HBM（大但慢）的读写次数
- **效果**：2~4 倍速度提升，显存从 O(N²) 降到 O(N)
- **数学等价**：结果和标准 Attention 完全一致，不是近似算法

### 4.4 SwiGLU MLP — 模型的"消化系统"

注意力交换完信息后，MLP 对每个词的信息做深度加工。

#### 数据流

```shell
输入 x (dim)
    │
    ├──→ w1 (dim → hidden_dim) ──→ SiLU 激活 ──┐
    │                                            ├──→ ×（逐元素相乘）──→ w2 (hidden_dim → dim) ──→ Dropout ──→ 输出
    └──→ w3 (dim → hidden_dim，无激活）────────┘
```

#### SwiGLU 公式

```shell
SwiGLU(x) = SiLU(x·W₁) ⊙ (x·W₃) · W₂
```

其中 SiLU 激活函数（也叫 Swish）：

```shell
SiLU(x) = x · σ(x) = x / (1 + e^(-x))
```

**通俗理解**：

- w1 负责**提取特征**，过 SiLU 后相当于给特征加了非线性开关
- w3 作为**门控（Gate）**，决定哪些特征该通过、哪些该拦住
- 两者逐元素相乘 → "门控筛选"：w1 和 w3 都认为重要的信息才保留
- w2 把维度压回 dim，方便与下一层对接

**hidden_dim 的计算**：

```shell
hidden_dim = round_up( (8/3) × dim,  multiple_of )
```

为什么是 8/3？SwiGLU 有三个权重矩阵（W1, W2, W3），相比标准 FFN 的两矩阵（W1, W2），参数量多了 50%。dim → 8/3 × dim 正好弥补这个差异，保持总参数量与标准 FFN 一致。

**例**：dim=768 → 8/3×768=2048 → 对齐到 64 的倍数 → 2048

### 4.5 DecoderLayer — "一层思考"

每一层 DecoderLayer 做两件事：看上下文 + 消化理解。

#### 结构

```shell
输入 x
  │
  ├──→ RMSNorm ──→ Attention ──→ + ──→ h    ← 子步骤1：看上下文（残差连接）
  │                        x ↗
  │
  └──→ RMSNorm ──→ MLP ──→ + ──→ 输出    ← 子步骤2：消化理解（残差连接）
                    h ↗
```

#### DecoderLayer 公式

```shell
h      = x + Attention( RMSNorm(x) )
output = h + MLP( RMSNorm(h) )
```

**Pre-Norm 设计**：先归一化再进子层（而不是先算再加再归一化），这是 LLaMA 的做法，训练更稳定。

**残差连接为什么重要？**

- 没有残差：网络只能"重写"信息，改错了就全毁了
- 有残差：网络只需学"修改量"（delta），原文的好内容不会丢
- 这让 12 层甚至 100 层的网络都能稳定训练

### 4.6 Transformer — 完整的大脑

把所有零件组装起来，得到一个完整的"语言模型"。

#### 结构总览

| 序号 | 组件 | 作用 |
| ------ | ------ | ------ |
| ① | `nn.Embedding` | 查表器：编号 → 向量 |
| ② | `nn.Dropout` | 随机丢弃，防过拟合 |
| ③ | `DecoderLayer × N` | 核心：N 层反复加工 |
| ④ | `RMSNorm` | 最终归一化 |
| ⑤ | `nn.Linear(dim, vocab_size)` | 输出每个字的得分 |

#### 参数共享（Weight Tying）

```python
self.tok_embeddings.weight = self.output.weight  # 输入查表和输出预测共用同一张权重表
```

**好处**：省了一半参数，而且实践证明效果反而更好（因为输入和输出的语义空间是对称的）。

#### 两种工作模式

**训练模式**（传了 targets）：

1. 输入完整句子 → 每个位置都预测下一个字
2. 用交叉熵算预测和正确答案的差距

\[
\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N} \log P(y_i | x_1, ..., x_{i-1})
\]

即对每个位置，模型猜对正确答案的概率取 -log，越接近 0 表示预测越准。

**推理模式**（不传 targets）：

1. 只看最后一个位置，输出对下一个字的预测分数
2. 取最高分的作为预测结果

#### 文本生成（generate）

自回归生成过程：

```shell
已有："今天天气"
    │
    ▼
① 喂给模型 → 输出每个字的得分
    │
    ▼
② 按 temperature 策略选下一个字（比如"真"）
   - temperature=0 → 直接选最高分（保守）
   - temperature>0 → 按概率随机抽（有创意）
   - top_k=n → 只从前 n 个候选里选
    │
    ▼
③ 把"真"接在后面："今天天气真"
    │
    ▼
④ 回到①，直到够长或遇到结束符
```

## 5. 分词器

分词器是模型和人类文字之间的"翻译官"。

```shell
人类文字 ──→ 分词器 ──→ 数字序列 ──→ 模型
模型输出 ──→ 分词器 ──→ 人类文字（解码）
```

### 算法：BPE（Byte-Pair Encoding，字节对编码）

BPE 像一个聪明的压缩算法：

1. 从基础字母表开始（256 个字节）
2. 扫描语料，找到出现最频繁的相邻字节对（比如 "的" + "了"）
3. 把这对合并成一个新 token
4. 重复上述过程直到词表满（比如 6144 个）

**效果**：

- 常见的词保留为独立 token："的"、"是"、"中国"
- 罕见字用字母/偏旁拼起来："鎏" 可能拆成两个 token
- 词表不大但什么都能表示

### 训练参数

| 参数 | 默认值 | 说明 |
| ------ | -------- | ------ |
| vocab_size | 6144 | 词典中 token 总数 |
| min_frequency | 2 | 出现少于 2 次的片段不收 |
| special_tokens | 5 个 | \<unk\>=0，\<s\>=1，\</s\>=2，\<\|im_start\|\>=3，\<\|im_end\|\>=4 |

### 对话模板

模型内置了聊天格式模板，支持多轮对话：

```shell
用户说："你好"
    ↓ 自动格式化
<|im_start|>user
你好<|im_end|>
<|im_start|>assistant
（模型开始生成回复）
```

## 6. 训练与推理流程

### 训练流程

```shell
原始文本数据 (JSONL)
    │
    ├──→ [分词器训练] ──→ tokenizer.json（词典）
    │
    └──→ 预处理（Tokenize 成数字序列）
              │
              ▼
         [模型初始化]
              │
              ▼
         循环每个 batch：
         ┌──────────────────────────────────┐
         │ ① 输入 token IDs → 模型 forward  │
         │ ② 输出 logits + 计算 loss（交叉熵）│
         │ ③ loss.backward()（反向传播）     │
         │ ④ optimizer.step()（更新参数）    │
         └──────────────────────────────────┘
              │
              ▼
         保存模型：config.json + model.safetensors
```

### 推理流程

```shell
用户输入（prompt）
    │
    ▼
分词器 → token IDs
    │
    ▼
自回归生成循环：
  model.generate(tokens, temperature=0.8, top_k=40)
    │
    ├── 模型 forward → 拿到最后一个位置的 logits
    ├── temperature 调整 + top_k 过滤
    ├── 按概率采样下一个 token
    └── 拼接到序列末尾 → 循环
    │
    ▼
分词器解码 → 返回生成文本
```

| `n_kv_heads` | 12 | 结构 | K/V 的头数（GQA，≤ n_heads） |
| `vocab_size` | 6144 | 结构 | 词表大小 |
| `hidden_dim` | None(自动) | 结构 | FFN 中间层维度，自动 = round_up(8/3×dim, multiple_of) |
| `multiple_of` | 64 | 结构 | FFN 维度对齐倍数 |
| `norm_eps` | 1e-5 | 正则 | RMSNorm 的 ε |
| `dropout` | 0.0 | 正则 | Dropout 比例 |
| `max_seq_len` | 512 | 推理 | 最大序列长度（上下文窗口） |
| `flash_attn` | True | 推理 | 是否使用 Flash Attention |

**参数量估算**（默认配置）：

| 组件 | 公式 | 参数数量 |
| ------ | ------ | ---------- |
| Embedding | vocab × dim | 6144 × 768 ≈ 4.7M |
| 每层 Attention | 4 × dim × dim | 4 × 768 × 768 ≈ 2.4M × 12 = 28.8M |
| 每层 MLP | 3 × dim × hidden_dim | 3 × 768 × 2048 ≈ 4.7M × 12 = 56.6M |
| 每层 RMSNorm | 2 × dim | 2 × 768 ≈ 1.5K × 12 = 18K |
| 最终 RMSNorm + Output | dim + dim×vocab | 768 + 4.7M ≈ 4.7M |
| 输出层 | 与 Embedding 共享 | 0 |
| **总计** | | **≈ 95M** |
